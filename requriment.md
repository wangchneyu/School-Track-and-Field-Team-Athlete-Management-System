# 校级田径训练管理与考勤可视化平台  
**需求说明书（V1.0）**

## 1. 项目概述

### 1.1 项目背景

校田径队日常训练与管理中存在以下问题：

- 考勤记录依赖纸质或口头统计，数据零散，难以追溯；
- 队员长期训练成绩、测试表现缺乏系统化记录与可视化；
- 教练对队员训练态度、表现的主观评价缺少结构化沉淀；
- 不同项目缺乏实时排行榜与数据驱动的选拔依据。

本项目拟开发一套基于 Python 的 Web 管理平台，为校级田径队提供“考勤记录、训练管理、成绩统计、教练评分与实时排行榜”一体化解决方案，实现队员与管理员（教练/队长）双端使用，支持长期数据保存与可视化展示。

### 1.2 建设目标

- 提供简单易用的日常考勤工具；
- 完整记录队员训练、测试成绩与主项表现；
- 实现教练对队员训练态度、综合表现的量化评分；
- 按项目生成实时成绩排行榜，辅助选拔与激励；
- 支持数据长期存储、导出备份及后续科研分析使用。

---

## 2. 用户角色与权限

### 2.1 角色定义

1. **管理员（Admin）**  
   一般为教练、助教、队长，拥有管理与配置权限。

2. **队员（Athlete）**  
   普通运动员账号，仅访问与自身相关及公开信息。

### 2.2 权限说明

**管理员：**

- 管理队员账号（创建、启用/禁用、查看信息）；
- 创建与管理训练场次（日期、时间、地点、备注）；
- 录入与修改考勤记录；
- 录入与修改成绩记录；
- 对队员进行训练表现评分与备注；
- 查看全队/分组统计数据（出勤、成绩、评分）；
- 查看所有项目排行榜；
- 导出相关数据（CSV/Excel）。

**队员：**

- 登录并查看个人信息；
- 查看个人历史考勤记录；
- 查看个人各项目成绩及变化趋势；
- 查看公开项目排行榜及自身排名。

## 3. 功能需求

### 3.1 身份认证与账号体系

1. **用户登录**
   - 使用账号 + 密码登录；
   - 登录后根据角色跳转至对应功能面板。

2. **账号管理（管理员）**
   - 创建队员账号：用户名、初始密码、姓名、学号、性别、训练组、主项；
   - 支持重置密码；
   - 支持账号启用 / 禁用。

> V1 可不对接统一身份认证，保留后续接入接口。

---

### 3.2 基础信息管理

1. **队员信息管理**
   - 字段示例：姓名、学号、性别、年级、训练组（短跑/中长跑/投掷/跳跃等）、主项、联系方式（可选）。
   - 管理员可按组别、项目查询与筛选。

2. **项目信息管理**
   - 字段示例：
     - 项目名：如 `100m`、`200m`、`跳远`、`铅球` 等；
     - 项目类型：`time`（越小越好）或 `distance`（越大越好）；
     - 性别限制（可选）。
   - 管理员可新增、编辑、禁用项目。

---

### 3.3 训练场次与考勤管理

#### 3.3.1 训练场次管理（管理员）

- 创建训练场次信息：
  - 日期；
  - 起始时间、结束时间；
  - 地点（可选）；
  - 训练内容备注（如“耐力跑”“专项力量”等）。
- 支持按日期范围、关键字查询历史场次。

#### 3.3.2 考勤记录管理

**录入方式（V1）：**

- 管理员在某场次详情页中，对全体队员进行状态标记：
  - `present`：出勤；
  - `late`：迟到；
  - `leave`：请假；
  - `absent`：缺勤；
  - 可填写备注（如“迟到 5 分钟”）。

**功能要求：**

- 每条考勤记录包含：训练场次、队员、出勤状态、记录时间、备注；
- 支持查询：
  - 按队员查看其全部历史考勤；
  - 按日期/场次查看整体出勤情况；
- 支持按条件导出（如按学期、按训练组导出）。

---

### 3.4 成绩记录与实时排行榜

#### 3.4.1 成绩录入（管理员）

录入内容包括：

- 队员；
- 项目；
- 成绩值（统一存为数值型，秒/米等）；
- 测试日期；
- 是否正式测试（如校运会、队内测验）；
- 备注（可选）。

#### 3.4.2 成绩查询

**队员端：**

- 查看个人各项目历史成绩列表；
- 查看单个项目成绩曲线（进步趋势）。

**管理员端：**

- 按项目、时间范围、训练组、是否正式测试筛选；
- 查看项目整体成绩分布情况。

#### 3.4.3 实时排行榜

系统从成绩数据中实时计算：

- 指定项目下，每名队员的**最佳成绩**（按项目类型判断优劣）；
- 项目类型为 `time`：数值越小名次越靠前；
- 项目类型为 `distance`：数值越大名次越靠前。

支持筛选：

- 性别；
- 训练组；
- 是否仅统计正式测试成绩。

展示字段示例：

- 排名、姓名、训练组、最佳成绩、最近一次成绩日期。

---

### 3.5 教练评分与综合评估

用于记录队员日常训练表现，供内部管理与选拔参考。

#### 3.5.1 初始评分项

建议包括（1–5 分制）：

- 出勤与纪律；
- 训练态度；
- 训练完成度 / 执行力；
- 文本备注。

#### 3.5.2 使用说明

- 管理员选择队员 + 日期进行评分；
- 支持查看历史评分记录；
- 默认仅管理员可见，后续可配置部分维度对队员开放。

#### 3.5.3 衍生分析（可选）

- 队员综合评分趋势；
- 各组平均评分对比；
- 作为内部评优、主力人选参考依据。

---

### 3.6 数据统计与可视化

#### 3.6.1 管理员可视化面板

展示内容示例：

- 出勤统计：
  - 全队 / 分组周、月出勤率折线图；
  - 迟到 / 缺勤占比饼图；
- 成绩统计：
  - 单项目成绩分布图；
  - 达标/优秀人数变化趋势；
- 个体视图：
  - 单个队员出勤率趋势；
  - 主项成绩进步曲线；
  - 综合评分趋势。

#### 3.6.2 队员个人面板

- 本月出勤日历视图；
- 主项成绩变化曲线；
- 自己在项目排行榜中的名次提示（如“当前 100m 男子第 3 名”）。

---

### 3.7 系统管理与操作日志

- 初始化超级管理员账号；
- 记录关键操作日志（如删除成绩、修改考勤、修改评分等），包含时间与操作者；
- 提供数据备份与恢复方案说明（具体实现视部署环境确定）。

---

## 4. 非功能需求

### 4.1 技术选型

- 开发语言：Python；
- 后端框架：推荐 FastAPI（或 Django / Flask，最终以实现团队为准）；
- ORM：SQLAlchemy；
- 数据库：
  - 开发环境：SQLite；
  - 正式环境：MySQL / PostgreSQL（推荐 MySQL）；
- 前端：
  - 初版：HTML + CSS + JavaScript + Bootstrap；
  - 可视化：ECharts / Chart.js；
- 部署环境：
  - Uvicorn / Gunicorn + Nginx；
  - 支持 Docker 容器化部署。

### 4.2 性能要求

- 面向 50–300 名队员规模；
- 常规查询接口响应时间控制在秒级；
- 支持分页查询，避免一次性加载过多数据导致卡顿。

### 4.3 安全要求

- 密码加密存储（哈希处理，如 bcrypt）；
- 严格的角色与权限控制；
- 防止普通队员访问、篡改他人数据；
- 对敏感操作（删除、修改关键数据）进行权限校验与日志记录。

### 4.4 易用性要求

- 界面简洁、中文本地化；
- 操作流程清晰，适配教练日常使用习惯；
- 支持搜索、筛选和导出功能，降低管理成本。

### 4.5 可扩展性要求

后续可扩展方向：

- 扫码签到（二维码 + 移动端 H5 页面）；
- 与学校统一身份认证对接；
- 消息通知（短信 / 企业微信 / 微信服务号）；
- 接入更多训练指标（心率、负荷、体测）用于科研与精细化管理。

---

## 5. 数据结构概要

核心数据表建议如下（具体以实现时的数据字典为准）：

- `users`：`id, username, password_hash, role`
- `athletes`：`id, user_id, name, student_id, gender, group, main_event, ...`
- `events`：`id, name, type`
- `sessions`：`id, date, start_time, end_time, description`
- `attendance`：`id, session_id, athlete_id, status, checkin_time, remark`
- `scores`：`id, athlete_id, event_id, performance, created_at, is_official`
- `ratings`：`id, athlete_id, coach_id, date, attitude, attendance, performance, comment`

---

## 6. 验收标准（V1）

以下条件满足视为 V1 版本通过验收：

1. 管理员可：
   - 管理队员账号与项目信息；
   - 创建训练场次并完成考勤记录；
   - 录入、编辑成绩记录；
   - 查看各项目实时排行榜。
2. 队员可：
   - 登录系统；
   - 查看个人基础信息；
   - 查看个人历史考勤与成绩；
   - 查看公开排行榜。
3. 数据能在数据库中完整持久化存储，支持导出；
4. 角色权限控制正确，未出现越权访问；
5. 在预期用户规模下系统运行稳定，无严重错误或数据丢失问题。
